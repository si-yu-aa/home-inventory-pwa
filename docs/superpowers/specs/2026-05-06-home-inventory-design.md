# 家庭库存 PWA 设计文档

## 目标

做一个轻量、适合 iPhone 使用的家庭库存 PWA，用来记录护肤品、彩妆、厨房调料、清洁用品等日常物品。它的核心价值是减少重复购买，并在物品已过期或即将过期时给出提醒。

## 产品范围

第一版是 local-first、单人自用版本。库存数据保存在浏览器本地，支持 JSON 备份和恢复，并且可以通过 iPhone Safari 添加到主屏幕。

App 支持手动录入、照片留存和纯文本批量录入。批量录入面向这样的工作流：用户拍摄柜子、抽屉或商品照片，让大模型按固定格式输出库存文本，再把文本粘贴到 app 里预览并导入。

第一版不做账号、云同步、多人协作和系统通知。这些能力以后可以作为独立模块追加。

## 数据模型

必填字段：

- 名称
- 分类
- 数量
- 单位

选填字段：

- 到期日
- 存放位置
- 购买日期
- 开封日期
- 品牌/规格
- 照片
- 备注

到期日不是必填。填写了到期日的物品会进入“已过期”和“30 天内到期”的提醒逻辑；没有到期日的物品只展示低优先级提示，不做强提醒。

## 架构

代码按职责拆成几个清晰模块：

- `domain`：共享类型、默认分类、默认位置、到期状态计算
- `features/inventory`：库存创建、更新、删除、本地存储、搜索、备份和恢复
- `features/bulkImport`：批量录入文本的解析和校验
- `pages`：页面级组合，例如首页、库存页、录入页和设置页；批量录入收在录入页内
- `components`：可复用的移动端 UI 组件
- `lib`：浏览器能力封装，例如照片压缩、文件下载、剪贴板复制

这样的边界让后续能力更好插拔。例如要加条码扫描、云同步、系统通知或更复杂的搜索，可以新增模块或扩展现有模块，而不是推翻重写。

## UI 设计

UI 采用 mobile-first 设计，优先保证 iPhone 竖屏好用。视觉方向是干净、生活化、轻量工具感，而不是后台管理表格。

主要界面：

- 首页：展示已过期、30 天内到期、未设置到期日和总库存概览
- 库存页：支持搜索、分类筛选、状态筛选和物品列表
- 录入页：提供“单个录入”和“批量录入”两个页内模式；单个录入把必填字段和选填字段分区展示，批量录入展示可复制的大模型提示词、统一存放位置输入、文本粘贴区、预览结果和错误
- 设置页：导出备份、导入备份、清空本地数据

## 搜索

库存搜索支持多关键词模糊匹配。用户可以用空格、英文逗号、中文逗号、顿号、分号或换行分隔关键词。

只要任意一个关键词命中物品的名称、分类、位置、品牌、备注或日期字段，就展示该物品。这个设计方便用户把大模型推荐的近义词一次性放进搜索框，例如：

```text
粉底 气垫、隔离
妆前乳
```

库存页还提供搜索扩写入口。用户先输入一个关键词，点击扩写按钮后，app 会复制一段让大模型生成近义词和相关叫法的 prompt，并尝试打开豆包 app。大模型返回的一行关键词可以再粘回搜索框。

## 批量录入格式

推荐格式：

```text
名称 | 分类 | 数量 | 单位 | 到期日 | 品牌规格 | 备注
洗面奶 | 护肤品 | 1 | 瓶 | 2026-11-30 | CeraVe 236ml | 已开封
生抽 | 厨房调料 | 2 | 瓶 | 2027-03-01 | 李锦记 |
散粉 | 彩妆 | 1 | 盒 |  | NARS | 无明确到期日
```

解析器接受有表头或无表头的文本。名称、分类、数量、单位必填；到期日如果填写，必须使用 `YYYY-MM-DD` 格式。

批量录入文本不包含存放位置。存放位置在页面上用“本批统一存放位置”单独填写，导入时合并到每条库存里。这样大模型输出更短，也更不容易把位置和品牌规格混在一起。

录入页的批量模式提供一个可复制的大模型提示词。提示词会要求大模型不要在文本里写存放位置。复制优先使用 Clipboard API；在 iOS 或非安全上下文中不可用时，退回到隐藏 textarea + `execCommand('copy')` 的兼容方案。
